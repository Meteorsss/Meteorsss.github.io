助手 · 回答

原因基本不在 “GPT 模型本身”，而在你给 Codex 的 自定义 provider + model catalog 里声明的能力。

我看了这两个文件：

  • C:\Users\30477\.codex\config.toml
  • C:\Users\30477\.codex\cc-switch-model-catalog.json

里面有几个关键点。

1. 为什么只能是高思考强度？

你的 config.toml 里写了：

model_reasoning_effort = "high"
model_catalog_json = "cc-switch-model-catalog.json"

然后 C:\Users\30477\.codex\cc-switch-model-catalog.json 里这个模型的配置是：

"default_reasoning_level": "high",
"supported_reasoning_levels": [
  {
    "description": "Disable Thinking",
    "effort": "none"
  },
  {
    "description": "Enabled Thinking",
    "effort": "high"
  }
]

也就是说，Codex 看到的能力声明是：

  • 默认思考强度：high
  • 支持的思考强度：只有 none 和 high
  • 没有声明 low
  • 没有声明 medium
  • 没有声明其他档位

所以 Codex UI 里不会给你显示其他思考强度。它不是根据模型名 gpt-5.5 自动猜能力，而是根据 model_catalog_json 里的 supported_reasoning_levels 来决定能不能选。

如果你希望能调节，需要 CC switch 生成/提供的 catalog 声明更多档位，例如类似：

"supported_reasoning_levels": [
  {
    "description": "Disable Thinking",
    "effort": "none"
  },
  {
    "description": "Low Thinking",
    "effort": "low"
  },
  {
    "description": "Medium Thinking",
    "effort": "medium"
  },
  {
    "description": "High Thinking",
    "effort": "high"
  }
]

并且：

"default_reasoning_level": "medium"

同时 config.toml 里也可以改成：

model_reasoning_effort = "medium"

但前提是：你的第三方 API / CC switch 后端真的支持这些 reasoning effort 参数。如果后端只接受 high 或者根本忽略 reasoning 参数，Codex 这边改了也可能无效，甚至请求报错。

2. 为什么不能上传图片？

你的 model catalog 里写的是：

"input_modalities": [
  "text"
],
"supports_image_detail_original": false

这等于告诉 Codex：

这个模型只支持文本输入，不支持图片输入。

所以 Codex 会禁用或隐藏图片上传/图片输入相关能力。

如果模型和第三方 API 确实支持视觉输入，catalog 至少需要把能力声明改成类似：

"input_modalities": [
  "text",
  "image"
],
"supports_image_detail_original": true

但这也只是让 Codex “认为”模型支持图片。真正能不能用，还取决于 CC switch 是否正确把 Codex 的图片输入转换成第三方 API 支持的格式。

3. 为什么不能上传文件？

这个也类似。你当前 catalog 没有声明文件输入能力，而且你的 provider 是：

[model_providers.custom]
name = "custom"
wire_api = "responses"
requires_openai_auth = true
base_url = "https://api.ggniao.com/v1"

你现在接的是第三方 OpenAI-compatible / Responses-compatible 接口。Codex 的文件上传、图片、工具调用等能力不是只看模型名，而是要看：

  1. Codex 本地 model catalog 是否声明支持；
  2. 自定义 provider 是否支持对应 API 形态；
  3. CC switch 是否实现了图片、文件、multipart、Responses input file 等转换；
  4. 第三方模型后端是否真正支持这些输入。

你当前 catalog 明确是文本模型：

"input_modalities": ["text"]

所以文件/图片能力被视为不可用。

4. 一句话总结

你现在 Codex 里之所以：

  • GPT 模型只能高思考强度；
  • 不能调低/中等思考强度；
  • 不能上传图片；
  • 不能上传文件；

主要是因为 C:\Users\30477\.codex\cc-switch-model-catalog.json 里把这个模型声明成了：

"text only"

并且 reasoning 只声明了：

"none" 和 "high"

Codex 是按这个能力表来渲染 UI 和构造请求的。

六、如果你还想“上传文件”

这里要分清楚两种“上传文件”。

情况 A：把文件作为 Codex 工作区文件处理

这种通常不需要模型 API 支持文件上传。
你把文件放进工作区，Codex 可以通过本地文件系统读取。

比如当前工作区：

C:\Users\30477\Documents\Codex\2026-07-07\new-chat-3

你可以把文件放在这里，然后告诉 Codex 文件路径。
这种方式依赖的是 Codex 本地文件读取能力，不是模型的 file input 能力。

情况 B：像 ChatGPT 那样把文件直接作为模型输入附件

这个就复杂一些。
Codex 的 custom provider + Responses wire API 要想完整支持“文件附件输入”，不仅要 catalog 声明，还要第三方 API / CC switch 正确兼容对应的 Responses 文件输入格式。

你当前 catalog 里没有看到明确的 file modality 字段。你可以尝试把：

"input_modalities": [
  "text",
  "image"
]

改成：

"input_modalities": [
  "text",
  "image",
  "file"
]

但我不保证 Codex 当前版本会识别 "file" 这个 modality，也不保证 CC switch 会正确转发。
更稳妥的做法是：把文件放到工作区，让 Codex 通过本地文件系统读取。
